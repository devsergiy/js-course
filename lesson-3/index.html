GIT

Git — распределённая система
управления версиями.

Система управления версиями (от англ. Version Control System, VCS или Revision Control System) — программное обеспечение для облегчения работы с изменяющейся информацией.

Такие системы наиболее широко используются при разработке программного обеспечения для хранения исходных кодов разрабатываемой программы.

Возможности CVS

  • Хранение несколько версий одного и того же документа
  • При необходимости возвращение к более ранним версиям
  • Определять, кто и когда сделал то или иное изменение

Репозиторий — Хранилище версионированных документов

Что значит распределенная CVS?

  • Каждый пользователь хранит копию основного репозитория
  • При необходимости любой пользователь может выступить в качестве зеркала
  • При сохранении изменений они переносятся в основную копию (origin)

Основные команды для синхронизации кода

Команда Назначение
Clone скопировать репозиторий себе на машину
Fetch получить удаленный изменения, но не применять их к локальной копии
Pull получить изменения и применить их к локальной копии
Commit сохранить снимок текущих изменений в файлах
Push отправить изменения в основной репозиторий

Команды для работы с ветками

Команда Назначение
Branch создать ветку или изменить ее свойства
Checkout Переключиться на ветку
Checkout -b переключиться на ветку с созданием ветки. Равносильно последовательным командам git branch, git checkout
Add пометить файлы для последующего сохранения
Commit сохранить изменения

Зачем нужны тесты?

В процессе разработки мы, время от времени, проверяем, правильно ли работает функция. Когда проверяешь работу кода вручную – легко его «недотестировать».

Пример:
  • Написали функцию f.
  • Тестируем с разными аргументами.
  • Вызов функции f(a) – работает, f(b) – не работает.
  • Поправили код – стало работать f(b)
  • Но не протестировали f(a) – которая возможно не работает.
Автоматизированное тестирование – это когда тесты написаны отдельно от кода, и можно в любой момент запустить их и проверить все важные случаи использования

Библиотеки для тестирования

Библиотека Назначение
Mocha содержит общие функции для тестирования, включая describe и it.
Chai поддерживает разнообразные функции для проверок.
Sinon для эмуляции и хитрой подмены функций «заглушками», понадобится позднее.
Jasmine объединяет в себе функции трех предыдущих, поэтому менее гибок
